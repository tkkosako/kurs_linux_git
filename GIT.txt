mkdir folder

git init folder  


git init - tworzenie repozytorium git, to jest miejsce gdzie są przechowywane pliki( powstaje folder .git)
git status - informuje nas co mamy w katalogu
git add linux.txt - dodajemy nasz plik do stagingu czyli zmian które czekają na dodanie do repozytorium. Gdy mamy więcej plików dodajemy je po koleii lub możemy wpisać je w jednej lini 
git commit -m " -commit czyli umieszczamy plik w naszym repozytorium ,przełącznik m oznacza że w cudzysłowiu będziemy dodawać jakąś etykietę"
git commit -m "initial" -> Komity zaczyna się od 
git log - historia naszych commitów
git log -p plik.txt - szczegółowe informacje zmian
git revert (6 znaków z commita) - cofnięcie zmian, dokonanych 

Git rejestruje zmiany w liniach. 

git checkout plik.txt
git reset - czysty staging

DODAWANIE REPOZYTORIUM ZDALNEGO
git remote add origin https://github.com/tkkosako/kurs_linux_git.git
origin - jego umowna nazwa
master - nazwa gałęzi
git push wysyłamy nasze repozytorium 
git push origin master
podajemy login i hasło

git push -u origin master

PODŁĄCZANIE SIĘ DO ISTNIEJĄCEGO JUŻ REPOZYTORIUM

git clone git@github.com:USER/REPO.git -> nie pyta o login i hasło, przydaje się do pracy juz na gotowym projekcie 
Gdy wykonamy git clone to git tworzy ukryty katalog gita w ściągniętym katalogu, trzeba wejść do niego i wtedy wykonać ściągnięcie czyli git pull
git pull origin master -> ściągnięcie z katalogu zdalnego
